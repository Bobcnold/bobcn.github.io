Laravel-基础-中间件

HTTP 中间件提供一个方便的机制来过滤进入应用程序的 HTTP 请求。 Laravel 框架已经内置一些中间件，包括维护、身份验证、CSRF 保护，等等。所有的中间件都位于 app/Http/Middleware 目录内。

Laravel 框架已经内置一些中间件，包括维护、身份验证、CSRF 保护，等等。所有的中间件都位于 app/Http/Middleware 目录内。

新建中间件

$ php artisan make:middleware NewMiddleware

NewMiddleware.php

在handle()方法中写入控制语句。

如符合条件，则可使用$next($request)语句跳过中间件。

<?php namespace App\Http\Middleware;

use Closure;

class OldMiddleware {

    public function handle($request, Closure $next)
    {
        if ($request->input('age') < 18)
        {
            return redirect('home'); // 重定向
        }
        return $next($request);
    }
}

Before / After 中间件

前置中间件：在请求前执行操作，如上例。

后置中间件：在请求后执行操作。

public function handle($request, Closure $next)
{
    $response = $next($request);

    // Perform action

    return $response;
}

注册中间件

全局中间件

所有的 HTTP 请求都执行。只要将中间件的类加入到 app/Http/Kernel.php 的 $middleware 属性清单列表中。

指派中间件给路由

需要先将中间件添加到 app/Http/Kernel.php 的 $routeMiddleware 列表中。

可终止中间件

为啥叫可终止中间件？

一旦定义了terminable中间件，则需要将其添加为全局中间件。

use Closure;
use Illuminate\Contracts\Routing\TerminableMiddleware;

class StartSession implements TerminableMiddleware {

    public function handle($request, Closure $next)
    {
        return $next($request);
    }

    public function terminate($request, $response)
    {
        // 相应已经被发送至客户端后的操作
    }
}
