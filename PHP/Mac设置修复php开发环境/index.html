Mac设置、修复php开发环境

  • 打开apache对php的支持
  • 设置apache多端口对应对目录
  • 拷贝php配置文件到/etc下，设置默认搜索路径
  • 修复mysql命令command not found
  • 修复php连接mysql报错"No such file or directory"

打开apache对php的支持

$ cd /etc/apache2
$ sudo vim httpd.conf

找到以下行，并去掉注释

LoadModule php5_module libexec/apache2/libphp5.so

测试apache配置（可在/Library/WebServer/Documents下添加index.php）

$ apachectl configtest

设置apache多端口对应对目录

$ cd /etc/apache2
$ sudo vim httpd.conf

找到Listen 80，在其下面添加行Listen 8080，监听其他端口

找到httpd-vhosts.conf所在行，打开其前面的注释

然后，去设置端口对应的目录

$ cd /etc/apache2/extra
$ sudo vim httpd-vhosts.conf

作如下设置：

<VirtualHost *:80>
ServerAdmin xcode_hu@163.com
DocumentRoot "/Library/WebServer/Documents/website_80"
ServerName bob-website-80
ErrorLog "/private/var/log/apache2/bob-website-80-error_log"
CustomLog "/private/var/log/apache2/dbob-website-80-access_log" common
</VirtualHost>

<VirtualHost *:8080>
ServerAdmin xcode_hu@163.com
DocumentRoot "/Library/WebServer/Documents/website_8080"
ServerName bob-website-8080
ErrorLog "/private/var/log/apache2/bob-website-8080-error_log"
CustomLog "/private/var/log/apache2/bob-website-8080-access_log" common
</VirtualHost>

重启apache即可

$ sudo apachectl restart

拷贝php配置文件到/etc下，设置默认搜索路径

$ sudo cp /etc/php.ini.default /etc/php.ini

修改权限

$ cd /etc
$ sudo chmod 644 php.ini

设置默认的搜索路径

; UNIX: "/path1:/path2"
include_path = "/Library/WebServer/Documents"

在php文件中可直接写成如下形式，相当于/Library/WebServer/Documents/fileName.php

<?php
    include('fileName.php');

修复mysql命令command not found

$cd ~
$sudo vim .bash_profile

在.bash_profile下方添加行

export PATH=${PATH}:/usr/local/mysql/bin

运行环境变量

$ source .bash_profile

无需每次重新打开终端重新source .bash_profile

$ vim ~/.zshrc

添加

export PATH=${PATH}:/usr/local/mysql/bin

运行

$ source ~/.zshrc

查看UNIX socket的值和phpInfo()页面中mysql.default_socket、mysqli.default_socket、pdo_mysql.default_socket的值是否相同。

打开php.ini，将上述三个字段的值改为UNIX socket的值。

mysql.default_socket = /tmp/mysql.sock

重启apache

$ apachectl restart
