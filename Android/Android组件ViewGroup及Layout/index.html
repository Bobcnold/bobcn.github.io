Android组件——ViewGroup及Layout

ViewGroup

ViewGroup是View的子类，主要作为视图容器类使用。它是一个抽象类，Layout都是它的子类。

ViewGroup容器控制其子组件的分布依赖于ViewGroup.LayoutParams、ViewGroup.MarginLayoutParams两个内部类。

  • ViewGroup.LayoutParams设置ViewGroup的位置信息。

  • ViewGroup.MarginLayoutParams设置ViewGroup边距信息（top、bottom、left、right等）。

Layout

子元素重要属性：

  • gravity：其内部的对齐方式。

  • layout_width/layout_height：match_parent、wrap_content。

LinearLayout-线性布局

orientation：horizontal或者vertical。

子元素重要属性

  • layout_gravity

    子标签在LinearLayout中的（相对于LinearLayout的）对齐方式。

    注意：当方向为horizontal时，横向的对齐方式无效；反之亦然。

  • layout_weight（权重）

    将剩余宽度按照比例分配。

RalativeLayout-相对布局

子元素重要属性
对齐align

  • 相对于父元素（即RalativeLayout）

    layout_alignParentTop / layout_alignParentBottom / layout_alignParentLeft / layout_alignParentRight

    layout_centerHorizontal / layout_centerVertical / layout_centerInParent

  • 相对于其他子元素

    layout_alignTop / layout_alignBottom / layout_alignLeft / layout_alignRight

相对（其它子元素）位置

layout_above / layout_below / layout_toLeftOf / layout_toRightOf

TableLayout-表格布局

TableLayout的orientation只能是vertical，设置为horizontal无效。

stretchColumns设置可被拉伸的列序号，shrinkColumns设置可被收缩的列序号，collapseColumns设置需要隐藏的列序号。多个列序号之间用逗号隔开。

子元素

TableLayout的每一个子元素占据一行。

子元素（TableRow及同级子元素）默认layout_width="match_parent"，可以不做设置，且设置为其他值无效；默认layout_height="wrap_content"，可不做设置，除<TableRow>子元素以外的其他子元素可做修改。

行（子）元素
<TableRow>
    <!-- 指定列序号 -->
    <TextView android:text="hello" android:layout_column="1" />
    <!-- 设置该单元格占据的列数 -->
    <TextView android:text="very well" android:layout_span="2" />
</TableRow>

FrameLayout-帧布局

子元素重要属性
  • layout_gravity：不同于LinearLayout，帧布局中可以同时指定横向和纵向的布局。

GridLayout-网格布局

Android4.0新增的布局方式。

对齐模式 行数量 列数量 保留行序号 保留列序号 使用默认页边距
alignmentMode rowCount columnCount rowOrderPreserved columnOrderPreserved useDefaultMargins
子元素重要属性
行序号 列序号 占据行数 占据列数 （相对于网格）对齐位置
layout_row layout_column layout_rowSpan layout_columnSpan layout_gravity

AbsoluteLayout-绝对布局

绝对布局。已经过时的布局方式，略。
