Laravel-基础-路由

学了一下路由，路由参数，命名路由，路由群组，模型绑定。

基础路由

Route::get('/', function()
{
    return "Hello World";
});

// 调用HomeController控制器的index()方法
Route::post('foo/bar', 'HomeController@index');

Route::match(['get', 'post'], '/', function()
{
    return view('welcome');
});

Route::any('foo', function()
{
    return "<h2>Hello Bob!</h2>";
});

路由参数

路由参数可以是可选择的。可以使用正则表达式限制参数。

路由参数不能包含-字符，使用_替代。

Route::get('/{id?}', function($id = 1)
{
    return view('welcome', ['id' => $id]);
});

Route::match(['get', 'post'], 'welcome/{name}', function($name)
{
    return "<h2>Hello World</h2>";
})
->where(['name' => '^\w+$']);

全局限制模式：如果希望特定的路由参数总是遵循特定的正则表达式，在RouteServiceProvider的boot方法里定义模式：

$router->pattern('id', '[0-9]+');

如果需要在路由外部获取其参数，可以

$id = $route->input('id');

或者使用Illuminate\Http\Request实体取得路由参数。

use Illuminate\Http\Request;

Route::get('user/{id}', function(Request $request, $id)
{
    if ($request->route('id'))
    {
        //
    }
});

命名路由

为路由命名，可以通过路由名跳转至该路由。

Route::post('user/profile', ['as' => 'disposeProfile', function()
{
    return view('test');
}]);

也可为控制器动作指定路由名称：

Route::get('user/profile', [
    'as' => 'profile', 'uses' => 'UserController@showProfile'
]);

可以使用路由名称产生URL或进行重定向

$url = route('profile');
$redirect = redirect()->route('profile');

currentRouteName方法会返回目前请求的路由名称：

$name = Route::currentRouteName();

路由群组

可以指定namespace（命名空间），prefix（URL前缀），middleware（中间件），domain（子域名路由）。

其中URL前缀可以设置路由参数，并且也可以通过正则限制参数（通过where）。

Route::group(['namespace' => 'Admin', 'prefix' => 'admin/{user_id}', 'where' => ['user_id' => '[0-9]+'], 'middleware' => ['foo', 'bar'], 'domain' => '{account}.myapp.com'], function()
{
    /*  URL: .../admin/23 
        controller路径: App/Http/Controllers/Admin/HomeController.php
     */
    Route::get('/', 'HomeController@index');

    Route::post('manage', function($account)
    {
        return view('user_manage', ['account' => $account]);
    });
});

定义全局模式

模型绑定

在 RouteServiceProvider::boot 方法定义模型绑定:

public function boot(Router $router)
{
    parent::boot($router);
    $router->model('user', 'App\User');
}

定义一个有{user}参数的路由，请求profile/1将注入ID为1的User实体。

Route::get('profile/{user}', function(App\User $user)
{
    //
});

未找到匹配的模型实体，则会返回404错误。
